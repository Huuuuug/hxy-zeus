# React + TypeScript + Vite

## 提交规范

```bash
git commit -m <type>[optional scope]: <description> //注意冒号后面有空格
- "type：提交的类型（如新增、修改、更新等）"
- "optional scope：涉及的模块，可选"
- "description：任务描述"
```


| 表头       | 表头                    |
|----------|-----------------------|
| feat     | 新功能                   |
| fix      | 修复bug                 |
| style    | 样式修改（ui校验）            |
| docs     | 文档更新                  |
| refactor | 重构代码（既没有新增功能也没有修复bug） |
| perf     | 优化相关，比如提升性能、体验        |
| test     | 增加测试，包括单元测试、继承测试等     |
| build    | 构建系统活外部依赖项目的更改        |
| ci       | 自动化流程配置或者脚本更改         |
| revert   | 回退某个commit提交          |
